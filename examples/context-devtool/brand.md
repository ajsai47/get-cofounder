# Brand

## Identity
- **Name**: Pipecheck
- **Tagline**: API testing that lives in your repo
- **Mission**: Make API testing as natural as unit testing — version-controlled, code-reviewed, and CI-native.

## Values
- **Code over clicks**: If it can't be expressed in a text file, it shouldn't be in the workflow. GUIs are for demos, not for real testing. Every test definition, every config, every environment lives in your repo.
- **Zero magic**: No hidden state, no implicit behavior, no "it works on my machine." Pipecheck reads a YAML file and does exactly what the file says. Predictable beats clever every time.
- **Community-first**: The CLI is open source and always will be. Community contributions are not an afterthought — they're core to the product. We build in the open, discuss in the open, and ship what the community needs.

## Personality
Pipecheck feels like a sharp senior engineer who writes great documentation and never overcomplicates things. Technical and precise, but not cold. Opinionated about good practices without being preachy. Quietly confident. The kind of tool that earns trust by being boring in the best way — it just works, every time.

## Visual Identity
- **Primary color**: #1E293B (slate-800) — dark, understated, terminal-native feel
- **Secondary color**: #22D3EE (cyan-400) — used for highlights, links, CLI output accents
- **Accent color**: #F59E0B (amber-500) — warnings, CTAs, attention-drawing elements
- **Background**: #FFFFFF (light docs) / #0F172A (dark docs and marketing)
- **Typography**: IBM Plex Sans for headings and body / IBM Plex Mono for code and CLI output
- **Imagery style**: Terminal screenshots with syntax-highlighted output. Architecture diagrams in monochrome with one accent color. No illustrations, no mascots. The product is visual enough on its own.

## Positioning
For backend developers and platform teams who test APIs, Pipecheck is the open-source CLI tool that lets you define, run, and automate API test suites in plain YAML, unlike GUI-based tools like Postman which can't be version-controlled, code-reviewed, or natively embedded in CI pipelines.

## Brand Don'ts
- Never use phrases like "developer experience platform" or "API lifecycle management" — we're a CLI tool, not an enterprise platform
- Never show GUI screenshots as the primary way to use Pipecheck — the terminal is the product
- Never claim to "replace" Postman — we solve a different problem (testing in code vs. ad-hoc exploration)
- Never use enterprise jargon: "stakeholders", "align", "digital transformation", "best-in-class"
- Never add a mascot or anthropomorphize the tool — keep it sharp and professional
