# Brand

## Identity
- **Name**: Shipfast
- **Tagline**: Ship your SaaS in days, not months
- **Mission**: Make it possible for any developer to launch a profitable SaaS product without a team.

## Values
- **Speed over perfection**: Ship the 80% solution. Real user feedback is worth more than hypothetical edge cases. We'd rather launch with rough edges than wait for polish.
- **Radical simplicity**: Every feature, every UI element, every line of copy should pass the "does this actually help the user?" test. If it doesn't, cut it.
- **Builder empathy**: We're developers building for developers. We know what it feels like to stare at a blank codebase at midnight. Everything we build should reduce that anxiety, not add to it.

## Personality
Shipfast feels like a confident friend who's shipped a bunch of products and is sharing what actually works. Not preachy, not academic, not corporate. Practical, slightly irreverent, and genuinely helpful. We make hard things feel approachable without dumbing them down.

## Visual Identity
- **Primary color**: #0F172A (slate-900) — dark, confident, developer-friendly
- **Secondary color**: #3B82F6 (blue-500) — action, links, interactive elements
- **Accent color**: #10B981 (emerald-500) — success, CTAs, positive signals
- **Background**: #FAFAFA (light mode) / #0F172A (dark mode)
- **Typography**: Inter for UI and body / JetBrains Mono for code
- **Imagery style**: Minimal, clean, lots of whitespace. Code snippets as hero images. No stock photos. Occasional developer humor in illustrations.

## Positioning
For solo developers who want to launch a SaaS product, Shipfast is the full-stack starter kit that handles auth, payments, emails, and deployment out of the box, unlike generic boilerplates which require days of configuration and leave you with a fragile foundation.

## Brand Don'ts
- Never use corporate buzzwords (synergy, leverage, ecosystem, scalable solution)
- Never promise "easy" — say "straightforward" or "simple." Building a business is hard. We make the technical parts simpler.
- Never talk down to the reader. They're smart developers. Respect that.
- Never use stock photography of people at computers
